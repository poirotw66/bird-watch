## 🎯 核心設計要點

### 1. 遊戲核心機制

**探索系統**
- 5 個棲息地：都市公園、森林、濕地、山區、海岸
- 動態環境：時間、天氣、季節變化
- 自由探索模式

**觀察機制**
- 望遠鏡：遠距觀察
- 相機：拍照記錄
- 錄音機：聲音記錄
- 筆記本：觀察筆記

**識別挑戰**
- 難度分級：新手、中級、專家
- 本地 vs 非本地物種
- 特徵記錄系統
- 即時回饋機制

### 2. 認知訓練設計

基於研究《The Tuned Cortex》設計的 5 種小遊戲：

1. **記憶配對** - 訓練工作記憶
2. **找不同** - 訓練精細識別
3. **快速識別** - 訓練反應速度
4. **聲音配對** - 訓練聽覺識別
5. **視覺搜索** - 訓練空間注意力

### 3. 進度系統

**玩家成長**
- 等級系統（1-50 級）
- 技能系統（觀察、識別、攝影、記憶）
- 經驗值與獎勵

**內容解鎖**
- 新棲息地
- 稀有鳥類
- 進階工具
- 特殊任務

### 4. 內容規劃

**鳥類資料**
- 常見留鳥：20 種
- 候鳥：10 種
- 特有種：10 種
- 總計：40+ 種

**任務系統**
- 教學任務：引導新手
- 主線任務：推進進度
- 支線任務：額外挑戰
- 每日任務：持續動機

**成就系統**
- 收集類成就
- 技能類成就
- 探索類成就
- 特殊成就

## 🏗️ 技術架構重點

### 核心技術選擇

```
語言: TypeScript 5.0+
建構: Vite 5.0+
渲染: HTML5 Canvas
音訊: Web Audio API
儲存: IndexedDB + LocalStorage
測試: Vitest
```

### 架構設計模式

**ECS 架構**
- Engine: 遊戲循環
- Scene: 場景管理
- GameObject: 遊戲物件
- Component: 組件系統

**事件驅動**
- EventSystem: 事件總線
- 解耦的系統通訊
- 靈活的擴充性

**模組化設計**
- 獨立的遊戲系統
- 清晰的介面定義
- 易於測試和維護

### 效能優化策略

1. **物件池模式** - 減少記憶體分配
2. **空間分割** - 優化碰撞檢測
3. **資源管理** - 延遲載入和快取
4. **渲染優化** - 離屏渲染和剔除

## 🎮 遊戲流程設計

### 新手體驗

```
開始遊戲 → 角色創建 → 教學引導 → 
第一次觀察 → 第一次識別 → 第一次記錄 → 
完成新手任務 → 解鎖新區域 → 自由探索
```

### 核心循環

```
探索棲息地 → 發現鳥類 → 觀察記錄 → 
識別挑戰 → 獲得獎勵 → 解鎖內容 → 
提升等級 → 探索新區域
```

### 長期目標

```
收集圖鑑 → 完成任務 → 解鎖成就 → 
訓練技能 → 挑戰高難度 → 成為大師
```

**採用敏捷開發**

**重視程式碼品質**

- 撰寫單元測試
- 進行程式碼審查

**注重使用者體驗**
- 簡潔直觀的介面
- 流暢的操作體驗
- 清晰的回饋機制

### 風險注意

**技術風險**
- Canvas 效能限制 → 考慮 WebGL
- 音訊同步問題 → 提前測試
- 跨瀏覽器相容 → 持續驗證

**內容風險**
- 資料收集困難 → 使用開放資源
- 素材品質不一 → 建立標準流程
- 版權問題 → 確保授權合法

**時程風險**
- 功能過於複雜 → MVP 優先
- 開發時間不足 → 分階段發布
- 測試時間壓縮 → 持續測試
